**表格列：** 工号、姓名、部门、职位、状态(在职/试用期/离职)、入职日期、合同到期日

**表格列：** 异动编号、员工姓名、类型、原部门→新部门、原职位→新职位、生效日期、状态

## 八、API 对接规范

### 8.1 请求规范

- 基地址：`/api`
- 认证方式：请求头 `X-User-Id`、`X-User-Name`、`X-Real-Name`
- 超时：`10000ms`
- Content-Type：`application/json`

### 8.2 响应规范

```json
{
  "code": 200,
  "data": {},
  "msg": "success"
}
```

### 8.3 状态码处理

| 状态码 | 处理 |
|--------|------|
| 200 | 正常处理 |
| 400 | 显示后端返回的 msg |
| 401 | 跳转登录页 |
| 403 | 显示权限不足提示 |
| 404 | 显示接口不存在提示 |
| 500 | 显示服务器错误提示 |

## 十、权限控制

### 10.1 菜单权限

| 菜单 | 权限码 | 说明 |
|------|--------|------|
| 用户管理 | ROLE_ADMIN | 仅管理员可见 |
| 人事管理 | ROLE_ADMIN | 仅管理员可见 |
| 系统设置 | ROLE_ADMIN | 仅管理员可见 |
| 其他菜单 | 已登录 | 所有登录用户可见 |

### 10.2 操作权限

- admin 用户：拥有所有操作权限
- 普通用户：只能操作自己创建/负责的数据
- 删除操作：需要弹窗确认
- 状态变更：需要记录操作日志
