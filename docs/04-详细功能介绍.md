## 一、系统管理模块

### 1.1 用户登录 [P0]

**功能描述：**
用户通过用户名和密码登录系统。登录成功后获取用户信息和权限列表，进入管理后台首页。

**界面规范：**
- 页面背景：主色渐变 `linear-gradient(135deg, #4f46e5, #7c3aed)`
- 登录框背景：`#fff`，圆角 `16px`，阴影 `0 20px 60px rgba(0,0,0,0.3)`，内边距 `40px`
- 系统名称：`22px`，字重 `600`，颜色 `#1f2937`
- 输入框高度：`42px`，圆角 `8px`，字号 `14px`
- 输入框边框：`1px solid #d1d5db`，Focus 时变为 `#4f46e5` + 光晕 `0 0 0 3px rgba(79,70,229,0.1)`
- 错误输入框边框：`#ef4444` + 光晕 `0 0 0 3px rgba(239,68,68,0.1)`
- 登录按钮：高度 `42px`，背景 `#4f46e5`，圆角 `8px`，字号 `15px`，字重 `600`，文字 `#fff`
- 登录按钮 Hover：背景 `#4338ca`
- 错误提示文字：`13px`，颜色 `#ef4444`

**交互规则：**
- 输入框为空时点击登录，显示错误提示"请输入用户名/密码"
- 密码错误时显示错误提示"用户名或密码错误"
- 登录中按钮显示 loading 状态，不可重复点击
- 登录成功后跳转到仪表盘页面

**API接口：**
- `POST /api/auth/login` - `{ username, password }` -> `{ token, user }`

## 三、考勤管理模块

### 3.1 考勤打卡 [P0]

**功能描述：**
员工通过 GPS 定位 + WiFi 校验进行上下班打卡。系统自动判断迟到/早退/正常状态。

**打卡流程：**
1. 获取 GPS 经纬度
2. 计算与公司的距离
3. 校验是否在允许半径内
4. 校验 WiFi MAC/名称是否匹配
5. 校验设备是否绑定（可选）
6. 判断打卡状态（正常/迟到/早退/迟到+早退/外勤）

**状态判定规则：**
- 签到时间 > 上班时间 + 弹性分钟 → 迟到(late)
- 签退时间 < 下班时间 → 早退(early)
- 签到迟到 + 签退早退 → 迟到+早退(late_early)
- 不在地理围栏内 → 外勤打卡(out_of_range)

**API接口：**
- `POST /api/attendance/clock-in` - 签到
- `POST /api/attendance/clock-out` - 签退
- `POST /api/attendance/unbind-device` - 解绑设备

---

### 3.2 考勤记录 [P0]

## 十一、费用报销模块

### 11.1 报销管理 [P1]

**功能描述：**
费用报销全流程管理，从申请到审批到财务核销。

**功能清单：**
- 创建报销（类型、标题、总金额、原因、明细列表）
- 报销列表（按用户、状态、类型筛选）
- 报销详情（含明细）
- 编辑报销（仅 pending/rejected 状态可编辑）
- 删除报销（软删除）
- 审批通过
- 审批拒绝（填写拒绝原因）
- 财务核销

**状态流转：**
```
pending(待审批) -> approved(已通过) -> verified(已核销)
pending(待审批) -> rejected(已拒绝) -> pending(重新提交)
```
