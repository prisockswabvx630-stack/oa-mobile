## 2. 部署准备

### 2.1 安装Node.js

```bash
# 使用NodeSource安装Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
node -v  # 应显示 v18.x.x
npm -v   # 应显示 9.x.x
```

### 2.2 安装MySQL

```bash
# Ubuntu安装MySQL
sudo apt-get update
sudo apt-get install mysql-server

# 启动MySQL服务
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置
sudo mysql_secure_installation

# 创建数据库
mysql -u root -p
```

```sql
CREATE DATABASE oa_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'oa_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON oa_system.* TO 'oa_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 2.3 安装Nginx

```bash
sudo apt-get install nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

### 2.4 安装PM2

```bash
npm install -g pm2
```

## 3. 后端部署

### 3.1 获取代码

```bash
# 克隆代码
git clone <repository_url> /var/www/oa-server
cd /var/www/oa-server/oa_server

# 安装依赖
npm install
```

### 3.2 配置环境变量

创建 `.env` 文件：

```bash
# 数据库配置
DATABASE_URL="mysql://oa_user:your_password@localhost:3306/oa_system"

# 服务端口
PORT=3000

# JWT密钥（如使用JWT认证）
JWT_SECRET="your_jwt_secret_key"
```

### 3.3 数据库迁移

```bash
# 生成Prisma客户端
npx prisma generate

# 执行数据库迁移
npx prisma db push

# 或使用迁移文件
npx prisma migrate deploy
```

### 3.4 初始化数据

```bash
# 运行初始化脚本（如有）
node scripts/init-data.js
```

### 3.5 启动服务

```bash
# 开发模式启动
npm run dev

# 生产模式启动
npm run build
npm start
```

### 3.6 使用PM2启动

```bash
# 使用PM2启动
pm2 start npm --name "oa-server" -- start

# 或使用ecosystem配置文件
pm2 start ecosystem.config.js
```

创建 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [{
    name: 'oa-server',
    script: 'dist/index.js',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    error_file: './logs/err.log',
    out_file: './logs/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss'
  }]
};
```

## 6. Nginx配置

### 6.1 创建配置文件

```bash
sudo nano /etc/nginx/sites-available/oa-system
```

### 6.2 配置内容

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名

    # PC端管理后台
    location / {
        root /var/www/oa-admin;
        try_files $uri $uri/ /index.html;
        
        # 缓存静态资源
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # API代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # API文档
    location /doc.html {
        proxy_pass http://127.0.0.1:3000;
    }

    # 安全配置
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_min_length 1000;
}

# HTTPS配置（推荐）
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    # SSL配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 其他配置同上...
}
```

### 6.3 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/oa-system /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启Nginx
sudo systemctl restart nginx
```

## 8. 数据库维护

### 8.1 备份数据库

```bash
# 创建备份脚本
cat > /var/www/backup-db.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/var/www/backups"
DATE=$(date +%Y%m%d_%H%M%S)
mysqldump -u oa_user -p'your_password' oa_system > "$BACKUP_DIR/oa_system_$DATE.sql"
# 保留最近30天的备份
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete
EOF

chmod +x /var/www/backup-db.sh
```

### 8.2 定时备份

```bash
# 添加定时任务
crontab -e

# 每天凌晨2点备份
0 2 * * * /var/www/backup-db.sh
```

### 8.3 恢复数据库

```bash
# 恢复备份
mysql -u oa_user -p'your_password' oa_system < /var/www/backups/oa_system_20260530_020000.sql
```

### 8.4 数据库优化

```
/var/www/oa-server/oa_server/logs/*.log {
    daily
    missingok
    rotate 30
    compress
    delaycompress
    notifempty
    create 0640 www-data www-data
    sharedscripts
    postrotate

    endscript
```

### 10.3 健康检查

创建健康检查脚本：

```bash
cat > /var/www/health-check.sh << 'EOF'
#!/bin/bash
response=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/api/health)
if [ $response != "200" ]; then
    echo "Service is down! Restarting..."
    pm2 restart oa-server
    # 发送告警邮件
    echo "OA Server restarted at $(date)" | mail -s "OA Server Alert" admin@example.com
fi
EOF

chmod +x /var/www/health-check.sh
```

添加定时检查：

```bash
# 每5分钟检查一次
*/5 * * * * /var/www/health-check.sh
```

## 11. 备份与恢复

### 11.1 完整备份

```bash
cat > /var/www/full-backup.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/var/www/backups"

# 备份数据库
mysqldump -u oa_user -p'your_password' oa_system > "$BACKUP_DIR/db_$DATE.sql"

# 备份上传文件
tar -czf "$BACKUP_DIR/uploads_$DATE.tar.gz" /var/www/oa-server/oa_server/uploads

# 备份配置文件
tar -czf "$BACKUP_DIR/config_$DATE.tar.gz" /var/www/oa-server/oa_server/.env

# 保留最近7天的备份
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete
EOF

chmod +x /var/www/full-backup.sh
```

### 11.2 恢复流程

```bash
# 1. 恢复数据库
mysql -u oa_user -p'your_password' oa_system < /var/www/backups/db_20260530_020000.sql

# 2. 恢复上传文件
tar -xzf /var/www/backups/uploads_20260530_020000.tar.gz -C /

# 3. 恢复配置文件
tar -xzf /var/www/backups/config_20260530_020000.tar.gz -C /var/www/oa-server/oa_server/

# 4. 重启服务
pm2 restart oa-server
```

## 12. 常见问题

### 12.1 服务启动失败

**问题**：PM2启动后立即退出

**解决方案**：
```bash
# 查看错误日志
pm2 logs oa-server --err

# 检查端口是否被占用
lsof -i :3000

# 检查环境变量
cat /var/www/oa-server/oa_server/.env
```

### 12.2 数据库连接失败

**问题**：无法连接到MySQL数据库

**解决方案**：
```bash
# 检查MySQL服务状态
sudo systemctl status mysql

# 检查数据库用户权限
mysql -u root -p
SELECT user, host FROM mysql.user;
SHOW GRANTS FOR 'oa_user'@'localhost';

# 测试连接
mysql -u oa_user -p'your_password' oa_system
```

### 12.3 Nginx 502错误

**问题**：访问网站显示502 Bad Gateway

**解决方案**：
```bash
# 检查后端服务是否运行
pm2 list

# 检查Nginx配置
sudo nginx -t

# 查看Nginx错误日志
tail -f /var/log/nginx/error.log

# 重启服务
pm2 restart oa-server
sudo systemctl restart nginx
```

### 12.4 静态资源404

**问题**：CSS/JS文件无法加载

**解决方案**：
```bash
# 检查文件是否存在
ls -la /var/www/oa-admin/

# 检查Nginx配置
cat /etc/nginx/sites-available/oa-system

# 重新构建前端
cd /var/www/oa-server/admin
npm run build
cp -r dist/* /var/www/oa-admin/
```

### 12.5 内存溢出

**问题**：Node.js进程内存溢出

**解决方案**：
```bash
# 增加Node.js内存限制
pm2 start npm --name "oa-server" --max-memory-restart 1G -- start

# 或在ecosystem.config.js中配置
max_memory_restart: '1G'
```
