## 二、后端服务部署 (oa_server)

### 2.1 开发模式启动

```bash
# 1. 进入项目目录
cd D:/sx1/oa_server

# 2. 安装依赖（首次运行）
npm install

# 3. 配置环境变量
# 编辑 .env 文件
DATABASE_URL="mysql://root:123456@localhost:3306/oa_system"
JWT_SECRET="your-secret-key"
PORT=3000

# 4. 初始化数据库（首次运行）
mysql -u root -p < oa_system_init.sql

# 5. 生成 Prisma Client
npx prisma generate

# 6. 启动开发服务
npm run dev
```

### 2.2 生产模式启动

```bash
# 1. 编译 TypeScript
npm run build

# 2. 启动服务
npm start
```

### 2.3 使用ts-node直接启动

```bash
cd D:/sx1/oa_server
npx ts-node src/index.ts
```

### 2.4 环境变量配置

编辑 `oa_server/.env` 文件：

```env
# 数据库连接
DATABASE_URL="mysql://root:123456@localhost:3306/oa_system"

# JWT密钥（生产环境请使用强随机字符串）
JWT_SECRET="oa-system-jwt-secret-2026"

# 服务端口
PORT=3000

# Node环境
NODE_ENV=development
```

### 2.5 访问地址

| 服务 | 地址 |
|------|------|
| API服务 | http://localhost:3000 |
| 健康检查 | http://localhost:3000/api/health |
| Prisma Studio | http://localhost:5555 (npx prisma studio) |

## 三、管理后台部署 (admin)

### 3.1 开发模式启动

```bash
# 1. 进入项目目录
cd D:/sx1/admin

# 2. 安装依赖
npm install
# 或
pnpm install

# 3. 启动开发服务器
npm run dev
# 或
pnpm dev
```

### 3.2 生产构建

```bash
# 1. 构建静态资源
npm run build

# 2. 预览构建结果
npm run preview
```

### 3.3 Nginx部署配置

```nginx
server {
    listen 80;
    server_name oa-admin.example.com;

    # 前端静态资源
    location / {
        root /var/www/oa-admin/dist;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 3.4 环境变量

编辑 `admin/.env.development`：

```env
VITE_API_BASE_URL=http://localhost:3000
VITE_APP_TITLE=智能OA管理后台
```

编辑 `admin/.env.production`：

```env
VITE_API_BASE_URL=https://api.oa.example.com
VITE_APP_TITLE=智能OA管理后台
```

## 七、常见问题

### 7.1 后端启动失败

**端口被占用**
```
Error: listen EADDRINUSE: address already in use :::3000
```
解决：
```bash
# 查找占用端口的进程
netstat -ano | findstr 3000
# 终止进程
taskkill /F /PID {进程ID}
```

**数据库连接失败**
```
Can't reach database server
```
检查：
1. MySQL服务是否启动
2. `.env` 中数据库连接信息是否正确
3. 数据库 `oa_system` 是否已创建
4. MySQL端口是否为3306

**Prisma Client未生成**
```
@prisma/client did not initialize yet
```
解决：
```bash
npx prisma generate
```

**依赖未安装**
```
Cannot find module
```
解决：
```bash
npm install
```

### 7.2 管理后台启动失败

**Vite版本不兼容**
```bash
# 清除缓存重新安装
rm -rf node_modules package-lock.json
npm install
```

**API请求失败**
检查 `vite.config.ts` 中的代理配置或 `.env` 中的API地址。

### 7.3 移动端启动失败

**Expo CLI未安装**
```bash
npm install -g expo-cli
```

**Metro bundler端口冲突**
```bash
npx expo start --port 8082
```

**Android模拟器无法连接本地API**
使用 `10.0.2.2` 替代 `localhost`（Android模拟器的特殊IP）。

## 八、生产环境部署方案

### 8.1 推荐架构

```
                    ┌──────────────┐
                    │   Nginx      │
                    │  (反向代理)   │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
     ┌────────┴──┐  ┌──────┴──────┐    │
     │ 管理后台   │  │  API服务    │    │
     │ Vue静态资源│  │  Node.js    │    │
     │ Nginx托管  │  │  PM2守护    │    │
     └───────────┘  └──────┬──────┘    │
                           │           │
                    ┌──────┴──────┐    │
                    │   MySQL     │    │
                    │  (主库)     │    │
                    └─────────────┘    │
                                       │
                              ┌────────┴──┐
                              │ 移动端App  │
                              │ APK/IPA   │
                              └───────────┘
```

### 8.2 PM2守护进程

```bash
# 安装PM2
npm install -g pm2

# 启动后端服务
cd D:/sx1/oa_server
pm2 start dist/index.js --name oa-server

# 查看状态
pm2 status

# 查看日志
pm2 logs oa-server

# 重启
pm2 restart oa-server

# 开机自启
pm2 save
pm2 startup
```

### 8.3 HTTPS配置

```nginx
server {
    listen 443 ssl;
    server_name oa.example.com;

    ssl_certificate /etc/nginx/ssl/oa.crt;
    ssl_certificate_key /etc/nginx/ssl/oa.key;

    location /api/ {
        proxy_pass http://localhost:3000;
    }

    location / {
        root /var/www/oa-admin/dist;
        try_files $uri $uri/ /index.html;
    }
}

server {
    listen 80;
    server_name oa.example.com;
    return 301 https://$host$request_uri;
}
```
