# 智能OA系统接口服务启动指南

## 前置要求

| 软件    | 版本   | 说明            |

| Node.js | 18+    | 必须安装        |

| npm     | 8+     | 随Node.js安装   |

## 启动方式

### 方式一：开发模式启动（推荐）

```bash
# 1. 进入项目目录
cd D:/sx1/oa_server

# 2. 安装依赖（首次运行）
npm install

# 3. 生成 Prisma Client
npx prisma generate

# 4. 启动开发服务（自动监听文件变化并重启）
npm run dev
```

### 方式二：生产模式启动

```bash
# 1. 编译 TypeScript
npm run build

# 2. 启动服务
npm start
```

### 方式三：直接 ts-node 启动

```bash
cd D:/sx1/oa_server
npx ts-node src/index.ts
```

## 停止服务

```bash
# 方法一：前台运行时按 Ctrl+C

# 方法二：查找并停止进程
netstat -ano | grep 3000
taskkill /F /PID {进程ID}

# 方法三：批量停止 Node 进程
taskkill /F /IM node.exe
```

## 常见问题

### 问题1：端口被占用

**错误信息：** `EADDRINUSE: address already in use :::3000`

**解决方法：**
```bash
netstat -ano | grep 3000
taskkill /F /PID {进程ID}
```

### 问题2：数据库连接失败

**错误信息：** `Can't reach database server`

**解决方法：**
1. 检查 MySQL 是否启动
2. 检查 `.env` 中数据库连接信息是否正确
3. 检查数据库 `oa_system` 是否已创建
4. 检查 MySQL 端口是否为 3306

### 问题3：Prisma Client 未生成

**错误信息：** `@prisma/client did not initialize yet`

**解决方法：**
```bash
npx prisma generate
```

### 问题4：依赖未安装

**错误信息：** `Cannot find module`

**解决方法：**
```bash
npm install
```
